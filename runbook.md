# Runbook — как запускать систему

Этот файл — главная инструкция для Claude и для тебя. Читается **первым** в любой новой сессии работы с этим проектом.

## Порядок загрузки контекста

В начале **любой** сессии генерации Claude читает файлы в этом порядке (порядок важен — поздние не должны затирать ранние):

1. `runbook.md` — этот файл (общий процесс)
2. `foundation/lessons.md` — **накопленные правила, перебивают всё остальное**
3. `foundation/strategist-system-prompt.md` — роль
4. `foundation/style-guide.md` — голос Ксении
5. `foundation/audience-decoded.md` — кому пишем
6. `foundation/offer-and-cta.md` — куда ведём
7. `foundation/threads-format.md` — в какой формат укладываем
8. Артефакты предыдущих шагов **текущего цикла** (если шаг ≥2)

Если эти файлы не прочитаны — не начинать генерацию. Сначала читать.

## Команды месячного цикла

Все команды формулируются простым русским языком. Claude по триггерным словам понимает, что делать.

### Шаг 1. Темы

> «Запусти цикл YYYY-MM, шаг 1» (или «сделай темы для YYYY-MM»)

**Что делает Claude:**
1. Создаёт папку `cycles/YYYY-MM/` если её нет.
2. Читает `audience-decoded.md`, `style-guide.md`.
3. Генерирует 30 тем — по одной на каждую фразу из `audience-decoded.md` (с возможностью группировки).
4. Каждая тема: формулировка + помеченный кластер аудитории + помеченный формат (соло / тред / эпизод).
5. Quality gate: покрыты все 5 кластеров (не менее 4 тем на каждый), нет смысловых дублей.
6. Пишет в `cycles/YYYY-MM/01-themes.md`.

### Шаг 2. Хуки

> «Запусти цикл YYYY-MM, шаг 2» (или «сделай хуки»)

**Что делает Claude:**
1. Читает `01-themes.md` и `threads-format.md`.
2. На каждую тему — **3 варианта первой строки** (≤12 слов каждая).
3. Каждый хук помечен эмоцией (узнавание / конфликт / стыд / любопытство).
4. Quality gate: длина ≤12 слов, нет шаблонов из антипаттернов («Знакомо?», «Сейчас расскажу…»).
5. Пишет в `cycles/YYYY-MM/02-hooks.md`.

### Шаг 3. Посты

> «Запусти цикл YYYY-MM, шаг 3» (или «сделай посты»)

**Что делает Claude:**
1. Читает `01-themes.md`, `02-hooks.md` (берёт лучший хук на каждую тему — или предлагает выбор).
2. На каждую тему — финальный текст:
   - **Соло** — один пост ≤500 знаков.
   - **Тред** — разметка `## Пост 1` … `## Пост N`, каждый ≤500 знаков.
   - **Эпизод** — короткая зарисовка-диалог, ≤500 знаков.
3. Структура — из `style-guide.md` (хук → разворот → финал-вопрос или сухая строка).
4. CTA — **только** в постах, помеченных `funnel-end` (см. шаг 4).
5. Пишет в `cycles/YYYY-MM/03-posts/NNN-<slug>.md` (NNN от 001 до 030).
6. Quality gate: длина ≤500 знаков, проверка по табу из `style-guide.md`, наличие CTA только в funnel-end.

**Шаги 3 и 4 идут параллельно** — Claude может сначала собрать воронки, а потом писать посты с пометкой funnel-end. Или сначала черновики постов, потом сборка воронок и финализация CTA. Решение — в момент запуска.

### Шаг 4. Воронки

> «Запусти цикл YYYY-MM, шаг 4» (или «собери воронки»)

**Что делает Claude:**
1. Читает `01-themes.md`, `offer-and-cta.md`.
2. Группирует темы в **4 цепочки по 5–7 постов** каждая.
3. Каждая цепочка: тема + последовательность (зацепил → углубил → инсайт → что можно → CTA) + назначенный CTA (CTA-1 или CTA-2).
4. По умолчанию: 3 цепочки на CTA-1 (онлайн-консультация), 1 цепочка на CTA-2 (Батуми). См. таблицу в `offer-and-cta.md`.
5. В каждой цепочке последний пост помечен как `funnel-end`.
6. Quality gate: обе CTA представлены, цепочки не наезжают одна на другую (одна тема — одна цепочка).
7. Пишет в `cycles/YYYY-MM/04-funnels.md`.

### Шаг 5. Календарь

> «Запусти цикл YYYY-MM, шаг 5» (или «сделай календарь»)

**Что делает Claude:**
1. Читает `04-funnels.md`, `02-hooks.md`.
2. Распределяет 30 единиц контента по 30 дням.
3. На каждый день: дата → пост (id) → к какой воронке относится → стадия (внимание / доверие / решение / действие) → краткое обоснование «почему сегодня».
4. Quality gate: подряд не идут 5+ однотипных хуков, обе воронки чередуются, finale-CTA-посты не стоят встык, в выходные — более лёгкий формат (зарисовки, личные истории).
5. Пишет в `cycles/YYYY-MM/05-calendar.md`.

## Команды для обратной связи

### Записать урок

> «Запиши в lessons: <свободный текст>»

**Что делает Claude:**
1. Понимает, к какому этапу относится (генеральное / темы / хуки / посты / воронки / календарь).
2. Понимает тип (NEVER / ALWAYS).
3. Переформулирует в формат `lessons.md` с датой и причиной.
4. Дописывает в нужную секцию `foundation/lessons.md`.
5. Кратко подтверждает: «записал в раздел X, формулировка такая-то».

### Ретро по циклу

> «Сделай ретро по YYYY-MM»

**Что делает Claude:**
1. Просит у тебя список постов, которые зашли / не зашли / переписала вручную перед публикацией.
2. По каждому случаю — гипотеза, что сработало или не сработало.
3. Из гипотез — кандидаты на новые правила в `lessons.md`.
4. Ты подтверждаешь — Claude записывает.

### Консолидация (раз в ~3 цикла)

> «Сделай консолидацию lessons»

**Что делает Claude:**
1. Читает весь `lessons.md`.
2. Находит дубликаты, схлопывает.
3. Находит правила, которые стали органикой (несколько циклов выполнялись без напоминания) — переносит в `## Архив` с датой переноса.
4. Показывает diff, ты подтверждаешь.

## Quality gates — общий принцип

Если автоматическая проверка валится — Claude **не отдаёт результат**, а возвращается и переделывает проблемные единицы. Только тогда показывает финал.

Конкретные проверки — в каждом шаге выше.

## Типовые операции (FAQ)

**«Перепиши пост 014 — слишком учительский тон».**
Claude читает `cycles/YYYY-MM/03-posts/014-*.md`, переписывает с опорой на `style-guide.md` (тёплый, рефлексивный, без императива), сохраняет в тот же файл. По ходу — если узнаёт паттерн — предлагает записать в `lessons.md`.

**«Дай мне темы только про самокритику»**.
Claude читает `audience-decoded.md` → кластер 1, генерирует только из него. Не сохраняет в `01-themes.md` (это разовый запрос), а отдаёт в чат.

**«Покажи, какие посты этого месяца ведут на Батуми».**
Claude читает `04-funnels.md`, выводит список постов с CTA-2.

**«Запусти цикл с нуля для следующего месяца».**
Подразумевает шаги 1→2→3→4→5 подряд для YYYY-MM. Между шагами Claude останавливается и показывает результат — ты подтверждаешь, потом следующий шаг.

## Что НЕ делает эта система

- Не публикует автоматически. Все посты — в файлах, ты вручную копируешь и публикуешь.
- Не отслеживает реакции из Threads (нет API в нашем сетапе). Аналитика — через ретро-шаг, на твоих наблюдениях.
- Не пишет статьи на 1500+ слов. Только Threads-формат.
- Не создаёт изображения. Если нужны — добавим отдельный шаг (через любой image-gen).
